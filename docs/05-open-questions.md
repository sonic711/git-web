# 待確認事項

這份文件只保留目前尚未定案、但會影響後續實作細節的議題。已確認項目已移入正式規格文件。

## 1. 共用部署模式

目前第一版以單人使用為主，但未來可能部署到一台共用機器。後續需再定義：

- 是否需要登入機制
- 是否需要操作紀錄對應到人員
- 是否允許不同使用者共用同一份設定檔

## 2. Repo 比對規則

目前規格要求：

- 若本機目錄已存在 repo，該 repo 必須與 `vendorRepoUrl` 相符

但仍需明確定義「相符」的判斷方式，例如：

- 完全字串相等
- 正規化後比對
- 只比對 `origin`，或允許其他 remote 名稱

## 3. Log 保留政策

目前已確認會寫 log，但尚未定義：

- 保留筆數
- 保留天數
- 是否要提供 UI 查詢歷史 log 列表

## 4. 排程格式

目前規格先採固定間隔排程，但後續仍需確認：

- 是否需要支援每日固定時間
- 是否需要支援每週固定時間
- 是否需要支援工作日排程

## 5. 共用機器上的併發保護

若未來部署到共用機器，需再定義：

- 同一筆 rule 是否允許重複觸發
- 是否要加檔案鎖或程序鎖
- 執行中 UI 是否要禁止再次點擊同步

## 6. 強制推送的進階策略

目前正式規格已確認使用者可透過 checkbox 觸發 `git push -f`。後續若要提升安全性，可再評估：

- 是否改為 `--force-with-lease`
- 是否加入二次確認視窗
- 是否限制只有特定規則可使用 force push

## 7. Review Gate 顯示深度

目前正式規格已要求 review-required rule 先看差異再同步，但尚未定義：

- commit-based push 是否限制為連續 commit
- 是否要支援多選非連續 commit
- 是否需要顯示檔案 diff 摘要
- 是否需要保存人工確認紀錄
